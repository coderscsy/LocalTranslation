# UI 设计说明

## 产品目标

界面优先服务三个高频动作：输入即译、框选即译、复制结果。应用始终展示离线模型状态，避免用户误以为内容会上传到网络。

## 参考与取舍

- Google Translate：采用源语言 / 目标语言双栏、居中的语言交换和输入后直接展示结果。
- DeepL for Windows：采用低干扰桌面工作流，把截图翻译和快捷键放在主操作附近。
- Microsoft Translator：采用明确的离线模型包状态和文本 / 图片模式入口。

只借鉴通用交互，不复制品牌、图标或视觉资产。

## 当前结构

```text
侧边导航
├── 实时翻译
├── 截图翻译
├── 翻译历史（待接入）
├── 本地模型状态（待接入管理器）
└── 设置（待接入）

主工作区
├── 实时翻译开关
├── 模型缺失提示
├── 源语言 / 目标语言 / 交换 / 截图
├── 原文输入 / 译文结果
└── 状态 / Ctrl + Enter / 立即翻译
```

## 交互约定

- 实时翻译默认开启，停止输入 650ms 后触发。
- `Ctrl + Enter` 强制执行一次翻译。
- `Ctrl + Shift + S` 交换源语言与目标语言。
- 截图、OCR、文本和译文默认只在内存中处理。
- 模型缺失、推理中、完成和失败状态必须可见。
